## Что это

Приложение MECHREVO Control Center поставляется только на **китайском и английском**. Этот проект добавляет полноценный **русский** интерфейс: переведены все строки, которые приложение читает из своих языковых файлов (610 строк — меню, настройки, питание, подсветка, дисплей, сведения о системе и т.д.).

## Совместимость

- **Приложение:** MECHREVO Control Center, версия **5.56.60.22** (пакет `CCU.WinUI`).
- **ОС:** Windows 10/11.
- На других версиях перевод, скорее всего, тоже встанет, но часть новых строк может остаться на английском (см. «Ограничения»).

## Установка

1. Скачайте `ru-ru.json`, `Install-Russian.bat` и `Uninstall-Russian.bat` в **одну папку**.
2. Запустите **`Install-Russian.bat`** (двойной клик).
3. Подтвердите запрос «Контроль учётных записей» (UAC) → **Да**.
4. Дождитесь надписи `DONE`.
5. Полностью закройте Control Center (в том числе значок в трее) и откройте заново.

Готово — интерфейс на русском. Язык в настройках приложения переключать **не нужно**, оставьте `en-us` (почему — ниже).

## Удаление

Запустите **`Uninstall-Russian.bat`** — он вернёт оригинальные файлы из резервных копий.

## После обновления приложения

Обновление перезапишет языковые файлы, и перевод слетит. Просто запустите **`Install-Russian.bat`** ещё раз.

## Как это работает

Приложение — это **упакованное WinUI-приложение (MSIX)**, установленное в защищённой папке
`C:\Program Files\WindowsApps\CCU.WinUI_5.56.60.22_x64__wrbgcf7aesyd8`.

Текст интерфейса оно берёт из двух источников:

1. **`Strings_Json\*.json`** — обычные текстовые файлы (UTF-8), по одному на язык. Приложение уже содержит пустую заготовку `ru-ru.json`. Мы её заполняем переводом.
2. **`resources.pri`** — скомпилированный бинарный индекс ресурсов (часть строк «вшита» сюда). С ним всё сложнее (см. «Ограничения»).

### Почему перевод пишется ещё и в `en-us.json`

Список языков в меню приложения **жёстко зашит** и для бренда MECHREVO содержит только `en-us` и `zh-cn` — добавить туда «Русский» нельзя без правки самой программы. Поэтому установщик кладёт перевод в два слота:

- `ru-ru.json` — на случай автоопределения языка системы;
- `en-us.json` — этот пункт **сейчас активен**, поэтому именно он отображается. После подмены приложение, формально оставаясь на «en-us», показывает русский.

### Почему нужны права администратора

Папка `WindowsApps` принадлежит системному аккаунту `TrustedInstaller`, и обычная запись туда заблокирована. Установщик повышает права, временно забирает владение нужным файлом (`takeown` + `icacls`), делает резервную копию и подменяет файл.

## Ограничения: что осталось на английском и почему

Часть строк (новые экраны: калибровка цвета, Mini LED, описания режимов ГП, Copilot, и др.) хранится **не в JSON, а в скомпилированном `resources.pri`**, и перевести их «на месте» нельзя по двум причинам:

1. **Кодировка.** Эти строки приложение читает как однобайтовую кодировку (Latin-1): 1 байт = 1 символ. Английские буквы (ASCII, байты 0–127) совпадают во всех кодировках и отображаются нормально. Русские буквы в Unicode имеют коды > 255 и в UTF-8 занимают 2 байта — однобайтовый «читатель» показывает каждый байт отдельным западным символом, получаются «кракозябры» (`ÐÐ°Ð»...`). В Latin-1 кириллицы нет в принципе, поэтому корректно вписать русский туда **физически невозможно**.
2. **Фиксированный размер.** Это бинарник с жёсткой вёрсткой: каждая строка лежит по фиксированному адресу, на неё ссылается таблица смещений. Заменить можно только на текст **той же длины в байтах**. Русский почти всегда длиннее (а в UTF-8 кириллица — по 2 байта на букву), поэтому длинные описания не помещаются, а удлинение «сдвигает» всё дальше и ломает файл.

> Аналогия: JSON — тетрадь, где пишешь что угодно и сколько угодно. `resources.pri` — уже отпечатанная книга: слово той же длины вписать можно, а заменить абзац более длинным — только перепечатав всю книгу.

«Перепечатать книгу» — это пересборка `resources.pri` через **`makepri`** (из Windows SDK): она собирает бинарник заново и снимает оба барьера. Мы от этого **отказались**, потому что:

- нужен Windows SDK (многогигабайтная установка);
- в приложении **нет** исходных файлов ресурсов (`.resw`) — только скомпилированный `.pri`, их пришлось бы восстанавливать вручную;
- высокий риск сломать приложение (потерять иконки/ресурсы), а проверить заранее нельзя.

Поэтому решение такое: **переводим всё, что лежит в JSON (это основной интерфейс), а строки из `resources.pri` оставляем на английском — но без «кракозябр».** Это лучший баланс «польза/риск».

В английском остаются также стандартные термины и бренды, которые обычно не переводят: `sRGB`, `P3`, `AdobeRGB`, `Mini LED`, `NVIDIA Optimus`, `Copilot`.

## Технические детали перевода

- Формат: плоский JSON `"ключ": "значение"`, UTF-8, та же структура ключей, что в оригинальном `en-us.json`.
- Объём: **610** строк, переведены все (0 строк осталось английскими внутри JSON).
- Технические сокращения и аббревиатуры (CPU, GPU, dGPU, IGPU, PL1/PL2, TGP, VBIOS, RGB и т.п.) сохранены как есть.

## Как внести вклад

Нашли кривой перевод, опечатку или хотите помочь — буду рада. Есть два пути.

**Сообщить об ошибке.** Откройте вкладку **Issues → New issue** и опишите, что не так (неверный или обрезанный перевод, ошибка установки). Укажите версию приложения (Настройки → О программе) и приложите скриншот, если можно.

**Прислать правку.** Для мелочи проще всего прямо на GitHub: откройте `ru-ru.json` → ✏️ → исправьте → **Propose changes** (создастся Pull Request). Для крупных правок — через форк:

```bash
git clone https://github.com/<вы>/mechrevo-control-center-ru.git
cd mechrevo-control-center-ru
git checkout -b fix/typo
# правите ru-ru.json
git commit -am "Fix: опечатка в strBatteryMode"
git push origin fix/typo
```

…затем на GitHub нажмите **Compare & pull request**.

**Что соблюдать:**

- меняйте только **значения** (текст справа); **ключи** слева (`"strBattery"`) не трогайте — по ним приложение находит строку;
- сохраняйте валидный JSON в **UTF-8 без BOM** и не удаляйте служебные `\n` и `\t`;
- технические термины (`CPU`, `GPU`, `dGPU`, `RGB`, `TGP`, `VBIOS`…) оставляйте как есть;
- строки из `resources.pri` перевести нельзя (см. «Ограничения») — их не присылайте.

Перед отправкой проверьте, что файл не сломан:

```bash
python -c "import json; json.load(open('ru-ru.json', encoding='utf-8')); print('OK')"
```

**Новая версия приложения.** Возьмите свежий `en-us.json` (`…\WindowsApps\CCU.WinUI_<версия>\Strings_Json\en-us.json`), добавьте и переведите новые ключи в `ru-ru.json`, не меняя их порядок, и пришлите PR с указанием версии.
